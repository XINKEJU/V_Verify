# 完全表单化解决方案：零下载、高流转效率方案

基于您的需求，我将提供一个完全无需用户下载App的解决方案，通过表单+H5+飞书多维表格实现高流转效率。

## 1. 整体架构设计

```
学生用户              → 微信H5表单页面
   ↓                         ↓
扫码/链接填写         → Supabase（主数据库）
   ↓                         ↓
数据自动处理         → 飞书多维表格（协同中心）
   ↓                         ↓
微信通知            → 联通工作人员（飞书）
   ↓                         ↓
处理结果             → 微信通知学生
```

// 同步到飞书多维表格
async function syncToFeishu(ticket) {
  const accessToken = await getFeishuAccessToken()

  const response = await fetch('https://open.feishu.cn/open-apis/bitable/v1/apps/:app_token/tables/:table_id/records', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${accessToken}`
    },
    body: JSON.stringify({
      fields: {
        '工单号': ticket.ticket_no,
        '手机号': ticket.phone,
        '姓名': ticket.name,
        '身份证号': maskIdCard(ticket.id_card), // 脱敏处理
        '学号': ticket.student_no,
        '需求类型': mapServiceType(ticket.service_type),
        '需求描述': ticket.description,
        '套餐': ticket.package || '',
        '状态': '待处理',
        '提交时间': ticket.created_at,
        '身份证正面': ticket.id_card_front_url ? [{ url: ticket.id_card_front_url }] : [],
        '身份证反面': ticket.id_card_back_url ? [{ url: ticket.id_card_back_url }] : []
      }
    })
  })

  return await response.json()
}

// 获取飞书访问令牌
async function getFeishuAccessToken() {
  const response = await fetch('https://open.feishu.cn/open-apis/auth/v3/tenant_access_token/internal', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      app_id: feishuAppId,
      app_secret: feishuAppSecret
    })
  })

  const data = await response.json()
  return data.tenant_access_token
}

// 脱敏身份证号
function maskIdCard(idCard) {
  if (!idCard || idCard.length < 15) return idCard
  return `${idCard.substring(0, 6)}********${idCard.substring(14)}`
}

// 映射服务类型
function mapServiceType(type) {
  const mapping = {
    'package_change': '套餐变更',
    'broadband_open': '开通宽带',
    'broadband_close': '关闭宽带',
    'query_account': '查询账号',
    'query_balance': '查询余额',
    'trouble_report': '故障报修',
    'cancel_account': '账户销户'
  }
  return mapping[type] || type
}

// 当工单状态变化时，自动发送通知
exports.onStatusChange = async (record, oldRecord) => {
  const newStatus = record.fields.状态
  const oldStatus = oldRecord.fields.状态
  const phone = record.fields.手机号

  if (newStatus !== oldStatus) {
    // 发送微信通知
    if (newStatus === '已派发联通') {
      await sendWechatMessage(phone, 
        `【校园网服务】您的需求已提交给联通处理，工单号：${record.fields.工单号}`
      )
    } else if (newStatus === '处理中') {
      await sendWechatMessage(phone,
        `【校园网服务】您的需求正在处理中，工单号：${record.fields.工单号}`
      )
    } else if (newStatus === '已完成') {
      const result = record.fields.处理结果 || '已完成'
      await sendWechatMessage(phone,
        `【校园网服务】您的需求已处理完成，结果：${result}，工单号：${record.fields.工单号}`
      )
    }

    // 在飞书中添加评论
    await addComment(record.record_id, 
      `状态更新：${oldStatus} → ${newStatus}，时间：${new Date().toLocaleString()}`
    )
  }
}

  if (newAssignee && newAssignee !== oldAssignee) {
    // 在飞书中@处理人
    await sendFeishuMessage(newAssignee[0].id,
      `您有新的工单需要处理：${record.fields.工单号} - ${record.fields.需求类型}`
    )
  }
}

## 安全加固与已知限制（2026-07-24 更新）

### 已修复（本次）
- **关闭 PII 泄露**：删除了 `anon` 对 `tickets` 表的 `SELECT` 全表可读策略。匿名角色现在无法直接读取任何工单。
- **安全查询通道**：新增 `SECURITY DEFINER` 函数 `get_my_tickets(p_phone)`，仅返回「该手机号自己的工单」，且不返回身份证号等敏感字段。前端查询页已改为调用此 RPC。
- **证件照直传 Storage**：提交时身份证照片由前端用已有的 `anon` 上传策略直传 `id-cards` 存储桶，并通过 `save_id_card_paths` RPC 回填路径，**不再依赖未部署的 Edge Function**。
- **频率限制**：新增 `submit_throttle` 表 + `check_submit_allowed` 函数，同一手机号 60 秒内最多提交 5 次，防刷单。
- **工单号防碰撞**：由 4 位随机改为「日期 + 时间戳36进制 + 3位随机」，基本杜绝 `ticket_no` 唯一约束冲突。
- **手机号不再明文进 URL**：提交页跳转查询页改用 `sessionStorage` 传递手机号。
- **条件化身份证要求**：仅 `package_change / broadband_open / close / trouble_report / cancel_account` 需要上传身份证；余额/账号查询无需上传。

### 仍需处理（需人工/本机操作）
1. **Edge Function 未部署**：`process-ticket` / `query-ticket` 仍停留在代码阶段。因当前网络环境对 Supabase 管理面（`api.supabase.com`）的 Cloudflare 封禁，无法在线部署。影响：飞书多维表格同步、邮件通知（Resend）暂未生效。核心提交/查询/证件照留存已不依赖它。
